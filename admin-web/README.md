# admin-web 使用说明

## 1. 这是什么

这是当前项目的 Web 后台。

它现在支持两种模式：

1. 本地模式（localStorage）
2. 云函数模式（直连 cloudfunctions/api）

可完成：

1. 员工录入
2. 项目录入
3. 日志查看
4. JSON 导入导出
5. 云端员工/项目拉取与上传

## 2. 怎么打开

最简单的方法：

1. 进入当前目录
2. 双击 index.html
3. 浏览器会直接打开后台页面

入口文件：

- index.html

## 3. 当前支持什么

1. 员工管理
2. 项目管理
3. 日志查看
4. 数据同步

## 4. 数据保存在哪里

默认保存在浏览器本地 localStorage。

在“数据同步”里配置 envId + adminKey 并连接成功后，可以和云函数双向同步。

这意味着：

1. 刷新页面不会丢
2. 换浏览器可能丢
3. 清理浏览器缓存会丢
4. 默认不是实时写入云数据库

## 5. 云函数直连怎么用

前置条件：

1. 云函数 `api` 已部署
2. 已创建 `users`、`tasks`、`audit_logs` 集合
3. 云函数环境变量已设置 `WEB_ADMIN_KEY`
4. 云开发身份认证已开启“匿名登录”（Web SDK 连接需要）

连接步骤：

1. 打开“数据同步”页面
2. 填写 envId
3. 填写 adminKey（即 WEB_ADMIN_KEY）
4. 点击“连接云函数”
5. 成功后可执行“从云端拉取”“上传到云端”

如果连接时报 `REFRESH_TOKEN_DISABLED`：

1. 打开云开发控制台当前环境
2. 找到“身份认证/认证设置”
3. 开启“匿名登录”
4. 刷新页面后重试

## 6. 仍推荐的稳妥使用方式

推荐这样使用：

1. 先在这里录员工和项目
2. 从“数据同步”页优先导出员工云导入 JSON 或项目云导入 JSON
3. 再导入微信云开发数据库

说明：

- 微信云数据库导入任务很多场景要求 JSON Lines 格式
- JSON Lines 就是一行一条 JSON 记录
- 但文件扩展名通常只接受 `.json` 或 `.csv`
- 当前后台导出的“云导入 JSON”就是这种内容为 JSON Lines、后缀为 `.json` 的文件
- 普通 JSON 更适合备份，不一定能直接导入数据库
