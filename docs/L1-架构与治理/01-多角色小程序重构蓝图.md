# L1-01 多角色小程序重构蓝图

更新时间：2026-03-27

## 目标

将当前项目重构为全小程序多角色系统，首批角色：

1. 管理者（领导/老板）
2. 执行者（配送/安装员工）
3. 财务（报销审核与记录）

并保留后续扩展角色能力。

## 当前阶段边界（MVP 1-2周）

本阶段要完成：

1. 登录后按角色进入不同入口
2. 页面级角色守卫
3. 云函数接口角色权限闸门
4. 管理者发布与审查最小闭环
5. 财务报销审核与记录最小闭环

本阶段暂不做：

1. 多级审批流
2. 复杂经营报表
3. 全量页面重命名与大规模UI翻新

## 角色职责

### 管理者

1. 查看项目情况、工人情况、运营情况
2. 每日发布项目
3. 审查项目执行结果

### 执行者

1. 接收任务并执行
2. 打卡与问题上报
3. 提交报销

### 财务

1. 审核员工报销
2. 记录审核结果
3. 支撑月度对账（二期增强）

## 成功标准

1. 三角色菜单和页面权限明确
2. 越权访问被后端拒绝
3. 关键操作可审计
4. 执行者主流程无回归
