# L3-01 重构执行SOP

更新时间：2026-03-27

## 适用范围

用于多角色小程序重构阶段的执行、验证与回滚。

## 执行步骤

1. 基线确认：部署当前云函数并记录版本
2. 文档先行：先更新 L1/L2，再做代码改动
3. 小步实施：每次只改一类能力（登录分流、页面守卫、后端权限）
4. 单批验证：每批完成后执行最小回归
5. 结果记录：记录成功项、失败项、风险项

## 每批提交清单

1. 代码改动
2. 对应文档改动
3. 回归结果
4. 回滚方式

## 回滚策略

1. 小程序端入口异常：回滚登录分流逻辑到默认执行者入口
2. 接口权限误拦截：回滚本批权限判断并保留审计日志
3. 文档与代码不一致：暂停发布，先对齐文档

## 最小回归（每批必做）

1. 执行者登录并进入工作台
2. 执行者接单、打卡、问题上报
3. 管理者可进入管理页且可查看项目
4. 财务可进入报销审核页
5. 非授权角色访问受限页面时被拦截
6. 云函数关键接口越权调用被拒绝
