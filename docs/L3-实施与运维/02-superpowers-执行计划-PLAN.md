# L3-02 Superpowers 执行计划 PLAN（1-2周）

更新时间：2026-03-27

## 工作流阶段（对应 superpowers skills）

1. brainstorming：冻结范围与验收标准。
2. writing-plans：拆解成可执行批次。
3. executing-plans：按批次实施代码与文档。
4. requesting-code-review：每批进行审查和缺陷归类。
5. verification-before-completion：执行回归与发布前验证。

## 执行批次

### 批次 A（第1-2天）角色基础设施

1. 新增角色与能力常量。
2. 登录后角色分流。
3. 页面守卫工具接入管理和财务相关页面。

交付物：

1. 代码改动。
2. L2 文档更新。
3. 回归记录。

### 批次 B（第3-5天）后端权限闸门

1. 管理与财务接口补充角色校验。
2. 修补高风险接口越权漏洞。
3. 写入关键操作审计日志。

交付物：

1. 接口权限矩阵。
2. 错误码与审计记录说明。
3. 回归记录。

### 批次 C（第6-8天）财务最小闭环

1. 报销页角色分视图。
2. 审核动作联调。
3. 历史记录可追溯。

交付物：

1. 前端页面改造。
2. 接口联调记录。
3. 回归记录。

### 批次 D（第9-12天）灰度与发布

1. 三角色端到端验证。
2. 小范围灰度。
3. 发布与回滚脚本确认。

交付物：

1. 发布记录。
2. 风险与问题清单。
3. 下一迭代 backlog。

## DoD（完成定义）

1. 代码、文档、回归记录三者齐全。
2. 每批变更可回滚。
3. 无 P0/P1 未关闭缺陷。
